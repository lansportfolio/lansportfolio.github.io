Technical SEO is the silent engine of digital marketing. No matter how compelling the content or aesthetic the design, a site will fail to convert if it suffers from slow load speeds or poor mobile responsiveness.

In this project, I performed a comprehensive technical audit of my e-commerce shop using Screaming Frog, Semrush, and Google Lighthouse. By optimizing crawlability and Core Web Vitals, I achieved a 98 Performance Score and ensured the site was optimized for both search engine bots and generative AI models.

Step 1: Crawlability & Indexing

The goal of this phase was to ensure that search engine crawlers could navigate the site efficiently, without encountering dead ends, crawl loops, or indexing barriers. To achieve this, I conducted a full technical audit using Screaming Frog, which led to the following optimizations:

Identify 404s

I identified several broken links caused by the mistakes when I initially built the site (It does happen sometimes), and resolved them to ensure all internal links functioned correctly.

Robots.txt

Then I added the robot.txt file following Google's robots.txt guidelines to tells bots where they aren't allowed to go, and make sure I did not accidentally block any site that should be indexable.

robot.txt

Sitemaps

I also generated an XML sitemap using Hexo plugins (as the site is built on Hexo). It automatically updates the sitemap every time I made new changes to the code, helping search engines discover new or updated pages efficiently.

sitemap.xml

GSC sitemap status

Canonicalization

To prevent duplicate content issues, I verified that every page includes a self-referencing canonical tag.

Canonical tag

Status Codes

Finally, I confirmed that all indexable pages return a 200 OK status code, rather than error responses such as 404 Not Found or 500 Server Error.

Screaming Frog auditing result

After completing these technical optimizations, I submitted the site to Google Search Console and Bing Webmaster Tools, and implemented IndexNow to encourage faster crawling and indexing. For more details, check out my blog post: How I Made My Website Crawlable for Search Engines

Step 2: Site Architecture

To me, the process of building a site architecture is like organizing important files: related content is grouped logically, nested within clear folders, and easy to locate when needed. This structure not only helps users navigate the site intuitively, but also allows search engines to efficiently crawl and understand the relationships between pages.

Site Hierarchy

Industry best practices consistently recommend a flat site hierarchy, where any page can be reached within 3 clicks or fewer from the homepage. Therefore, I reviewed my site structure and ensured it followed this principle to improve both usability and crawl efficiency.

Screaming Frog auditing result

Navigation

  • Navigation bar: Designed a clean, intuitive navigation bar that remains consistently visible, allowing users to access key pages at any point.
  • Breadcrumbs: Implemented breadcrumb navigation on all product pages to help users move easily between category levels.

Navigation bar and breadcrumbs

URL Structure

I optimized URLs to be:

  • Descriptive and readable: using hyphens ( - ) rather than underscores ( _ )
  • Clean and concise: avoiding unnecessary parameters or excess depth

Internal Linking

Since internal linking is part of my content creation strategy, I mostly inserted internal links between my product pages and blog post pages to signal topical relevance to search engines and distribute link equity across the site.

  • link URLs: Fixed one broken internal link in blog post page
  • Anchor text: Optimized anchor text to be relevant, descriptive, and contextually meaningful

Multiple URL Versions

During development, the site was hosted on GitHub Pages, which automatically generated a .github.io URL. After purchasing a custom domain, I configured a permanent 301 redirect to ensure all traffic resolves to the primary .com domain to prevent duplicate content and consolidate authority.

Step 6: Performance & Core Web Vitals

This was the most critical phase. Since Google Search Console requires high traffic for data, I used Google Lighthouse to simulate real-world performance and generate a comprehensive performance audit, identifying opportunities for improvement across the core metrics Google evaluates.

The Challenge: Initial Performance Bottlenecks

My initial report showed a sub-optimal score due to high Largest Contentful Paint (LCP) and First Contentful Paint (FCP).

  • JS Rendering Impact: A "fade-in" animation on the homepage was delaying the LCP by over 3200ms.
  • Unoptimized Assets: High-resolution images were causing significant payload issues.

Lighthouse score for Desktop (Before)

Lighthouse score for Mobile (Before)

The Solution: Optimization Tactics

  • CSS Modification: I removed the JavaScript-based animation of homepage headlines, drastically reducing the main-thread work.
  • Asset Compression: The audit revealed that images accounted for 1690 KiB of cache usage on the homepage. So I reduced file sizes through compression and ensured images were properly sized for their display dimensions.
  • Code Minification: I also used AI-assisted tools (Cursor) to minify JavaScript and CSS files, stripping unnecessary characters to reduce file size.

Utilized AI-tools for code optimization

The Result: A "Near-Perfect" Score

After these iterations, the site's homepage reached a 98/100 Performance Score, with an LCP of under 1.0 seconds - well within the "Good" threshold for Google.

Summary

For technical SEO optimizations, I focused on AI-Readiness. By providing a perfectly structured, light and fast foundation, I ensured that search engines and AI agents can easily parse, trust, and cite my content.

My key takeaways in this project:

  1. Performance is User Experience: I realized that "speed" isn't just a number for bots; it’s a psychological barrier. Reducing LCP directly correlates with a lower bounce rate and higher user trust.
  2. The "Fix it in the Code" Mentality: Using a static site generator like Hexo taught me that the most effective SEO is often solved at the code level, not the CMS level. Minifying files and optimizing CSS is just as important as writing a good meta description.
  3. Iterative Auditing: SEO is not a "set and forget" task. Using tools like Lighthouse throughout the development process allowed me to catch performance regressions before they went live.
